# Sensitive and meta prompts

**Parent:** [`README.md`](../README.md)

## What this probes

Questions that are **easy to answer badly**: meta (weakness, why leave), ethics theater, manager conflict, DEI without substance, or experiences many ICs **never had**. Plan **short boundaries** + **nearby** stories where the old prep already did the work.

## Example questions

| # | Prompt | Why it can be a **bad-answer** trap |
| --- | --- | --- |
| 1 | Tell me about a time you **disagreed with your manager**. | Sounds insubordinate if poorly framed. |
| 2 | Describe an **ethical dilemma** or pressure to cut corners. | Preachy or TMI. |
| 3 | Tell me about a time you **missed a deadline**. | Fake “we all learned” is obvious. |
| 4 | Tell me about **PIP / letting someone go / performance conversation**. | Many ICs have **no** story—OK. |
| 5 | Tell me about **DEI** on a team. | Empty virtue signaling fails. |
| 6 | **Why are you leaving?** / **Why this company?** | Not STAR; trashing employers fails. |
| 7 | **What is your biggest weakness?** | Clichés fail. |
| 8 | Tell me about **failing a customer or losing trust**. | High honesty bar. |
| 9 | Tell me about a time you **said no**. | Customer vs manager differs. |
| 10 | Tell me about working with someone **difficult**. | Can sound petty. |
| 11 | **Most complex system**—explain to a **non-engineer**. | Rambling fails. |
| 12 | **Gaps in your resume** / short tenure / employment gap. | Brief, factual. |
| 13 | Tell me about **compensation** expectations or competing offers. | Often boundary + redirect. |
| 14 | Tell me about **conflict with skip-level** or exec. | Rare; stay factual and short. |

## Possible anchors (or “none”)

| Prompt | Possible anchors (or “none”) |
| --- | --- |
| Disagreed with manager | Prefer **data-led** dissent + **commit after decision**: [load testing gate](../../work-experience/2021-amazon-fba.md#making-load-testing-a-mandatory-quality-gate) or capacity framing; if **never**, say so and use **peer/TL** disagreement ([Python SDK throughput](../../work-experience/2022-orkes-sdk.md#improving-python-sdk-throughput-with-data-driven-concurrency-decisions)). |
| Ethical dilemma / cut corners | Often **no perfect story**—**process** answer: code review, escalation, stop-the-line; or [safe testing without exchange](../../work-experience/2019-beyond-hft.md#building-a-safe-testing-environment-without-exchange-access) as **risk-aware** behavior. |
| Missed deadline | If you **lack** a good example: admit **narrowly** + **what changed** (buffer, earlier comms). [Invoice deadlines](../../work-experience/2021-amazon-fba.md#delivering-seller-facing-invoice-improvements-under-tight-deadlines) **only** if truth matches—**do not** claim miss if you did not miss. |
| PIP / firing / performance | “I have not owned a formal performance process; I *have* given **direct feedback** in 1:1s” + redirect, if closest fit: [C++ onboarding / expectations](../../work-experience/2019-beyond-hft.md#accelerating-onboarding-in-a-large-c-codebase) or [structured handoff under capacity pressure](../../work-experience/2022-orkes-sdk.md#clojure-sdk-calendar-limits-and-a-mentoring-handoff)—**not** the same as running a PIP. |
| DEI | Only **real** examples. If none: boundary + **inclusive code review and clear docs** + [documentation friction](../../work-experience/2022-orkes-sdk.md#reducing-sdk-onboarding-friction-through-code-driven-documentation) if authentic. |
| Why leaving / why here | Short **depth A/B/C** options + Brazil/health spine: [current situation — Q&A lines](../../work-experience/current-situation.md#-public-qa-lines); still keep **two forward sentences** in live answers, no bitterness. |
| Weakness | Habits not traits; e.g. underestimated cross-team latency + [catalog dependency](../../work-experience/2024-aws-gbl.md#unblocking-large-scale-catalog-classification-under-cross-team-dependencies) as what you changed. |
| Failed customer / trust | If **no** strong story: boundary + closest **pain reduction** → [on-call noise](../../work-experience/2021-amazon-fba.md#reducing-on-call-noise-through-root-cause-fixes) or [SDK stability](../../work-experience/2025-aws-connect.md#stabilizing-a-high-risk-public-sdk-release-with-proactive-dependency-alignment). |
| Said no | [Load testing gate](../../work-experience/2021-amazon-fba.md#making-load-testing-a-mandatory-quality-gate); or [profile page](../../work-experience/2024-aws-gbl.md#delivering-a-new-public-user-profile-page-as-a-full-stack-owner) if scope negotiation is true. |
| Difficult person | **Behavior** not character: conflicting priorities / slow responses + [catalog](../../work-experience/2024-aws-gbl.md#unblocking-large-scale-catalog-classification-under-cross-team-dependencies). |
| Non-engineer explanation | One mental diagram from [Connect](../../work-experience/2025-aws-connect.md) or [Beyond](../../work-experience/2019-beyond-hft.md); **30 seconds**, no jargon ladder. |

## Gaps and detail drills

- Keep [rehearsal boundary table](../rehearsal.md#when-you-may-not-have-a-good-answer) in muscle memory.
- **2017 Algar:** [role file](../../work-experience/2017-algar-telecom.md)—thin on behavioral **###** stories; many early-career questions may have **no strong STAR** there. Prefer being **honest** over inventing drama.
